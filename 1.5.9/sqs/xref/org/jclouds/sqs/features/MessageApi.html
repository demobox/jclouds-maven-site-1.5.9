45     /**
46      * The DeleteMessage action deletes the specified message from the specified
47      * queue. You specify the message by using the message's receipt handle and
48      * not the message ID you received when you sent the message. Even if the
49      * message is locked by another reader due to the visibility timeout setting,
50      * it is still deleted from the queue. If you leave a message in the queue
51      * for more than 4 days, SQS automatically deletes it.
52      * 
53      * <h4>Note</h4>
54      * 
55      * The receipt handle is associated with a specific instance of receiving the
56      * message. If you receive a message more than once, the receipt handle you
57      * get each time you receive the message is different. When you request
58      * DeleteMessage, if you don't provide the most recently received receipt
59      * handle for the message, the request will still succeed, but the message
60      * might not be deleted.
61      * 
62      * <h4>Important</h4>
63      * 
64      * It is possible you will receive a message even after you have deleted it.
65      * This might happen on rare occasions if one of the servers storing a copy
66      * of the message is unavailable when you request to delete the message. The
67      * copy remains on the server and might be returned to you again on a
68      * subsequent receive request. You should create your system to be idempotent
69      * so that receiving a particular message more than once is not a problem.
70      * 
71      * @param queue
72      *           the queue the message is in
73      * @param receiptHandle
74      *           The receipt handle associated with the message you want to
75      *           delete.
76      */
77     void delete(String receiptHandle);

113    /**
114     * The ChangeMessageVisibility action changes the visibility timeout of a
115     * specified message in a queue to a new value. The maximum allowed timeout
116     * value you can set the value to is 12 hours. This means you can't extend
117     * the timeout of a message in an existing queue to more than a total
118     * visibility timeout of 12 hours. (For more information visibility timeout,
119     * see Visibility Timeout in the Amazon SQS Developer Guide.)
120     * 
121     * For example, let's say the timeout for the queue is 30 seconds, and you
122     * receive a message. Once you're 20 seconds into the timeout for that
123     * message (i.e., you have 10 seconds left), you extend it by 60 seconds by
124     * calling ChangeMessageVisibility with VisibilityTimeoutset to 60 seconds.
125     * You have then changed the remaining visibility timeout from 10 seconds to
126     * 60 seconds.
127     * 
128     * <h4>Important</h4>
129     * 
130     * If you attempt to set the VisibilityTimeout to an amount more than the
131     * maximum time left, Amazon SQS returns an error. It will not automatically
132     * recalculate and increase the timeout to the maximum time remaining.
133     * 
134     * <h4>Important</h4>
135     * 
136     * Unlike with a queue, when you change the visibility timeout for a specific
137     * message, that timeout value is applied immediately but is not saved in
138     * memory for that message. If you don't delete a message after it is
139     * received, the visibility timeout for the message the next time it is
140     * received reverts to the original timeout value, not the value you set with
141     * the ChangeMessageVisibility action.
142     * 
143     * @param queue
144     *           the queue the message is in
145     * @param receiptHandle
146     *           The receipt handle associated with the message whose visibility
147     *           timeout you want to change. This parameter is returned by the
148     *           ReceiveMessage action.
149     * @param visibilityTimeout
150     *           The new value for the message's visibility timeout (in seconds)
151     *           from 0 to 43200 (maximum 12 hours)
152     */
153    void changeVisibility(String receiptHandle, int visibilityTimeout);

361    /**
362     * The ReceiveMessage action retrieves one or more messages from the
363     * specified queue. The ReceiveMessage action does not delete the message
364     * after it is retrieved. To delete a message, you must use the DeleteMessage
365     * action. For more information about message deletion in the message life
366     * cycle, see Message Lifecycle.
367     * 
368     * <h4>Note</h4>
369     * 
370     * Due to the distributed nature of the queue, a weighted random set of
371     * machines is sampled on a ReceiveMessage call. That means only the messages
372     * on the sampled machines are returned. If the number of messages in the
373     * queue is small (less than 1000), it is likely you will get fewer messages
374     * than you requested per ReceiveMessage call. If the number of messages in
375     * the queue is extremely small, you might not receive any messages in a
376     * particular ReceiveMessage response; in which case you should repeat the
377     * request.
378     * 
379     * @param queue
380     *           from where you are receiving messages
381     * @return message including the receipt handle you can use to delete it
382     */
